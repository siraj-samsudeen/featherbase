# Roadmap

Capabilities in build order. Each gets its own `docs/capabilities/N-name/` folder (research → spec → plan) written just-in-time — specs written far ahead of build are fiction. Later entries are provisional by design.

| #   | Capability                                                                                                                                                                                                                                                                                       | The tracer-bullet test                                                                                                        |
| --- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------- |
| 1   | **Scaffold** — monorepo, React + Vite + TanStack Router, Convex, feather-testing-convex wired in, CI _that runs tests_                                                                                                                                                                           | A seeded integration test passes in CI                                                                                        |
| 2   | **DocType engine** — metadata tables, per-DocType record tables + fieldIndex sidecar, repository layer, package-mode codegen (schema.ts + types + hook stubs), materialization ladder                                                                                                            | Test matrix covers CRUD + filter/sort on user-defined fields at realistic row counts; promotion round-trip is property-tested |
| 3   | **Auto-generated UI** — create a DocType, get working grid, form, and detail views. Grid library needs re-research first — Glide Data Grid is unmaintained on npm since 2024-02 ([revalidation](research/revalidation-2026-07.md)); evaluate maintained forks, AG Grid, TanStack Table + Virtual | Build a working app with zero code                                                                                            |
| 4   | **Excel import** — drop a file → inferred schema (types, enums, relation suggestions) → preview → approve → DocType + data                                                                                                                                                                       | A real handed-over spreadsheet becomes a working app                                                                          |
| 5   | **Permissions** — roles + one-toggle Row Owners in the repository layer                                                                                                                                                                                                                          | Multi-user isolation tests pass                                                                                               |
| 6   | **Flows engine** — interpreter over `@convex-dev/workflow`; triggers (doc events, cron, button); Action/Loop/Condition grammar; run history with per-step payloads                                                                                                                               | Fake-timer tests fast-forward sleeps and retries                                                                              |
| 7   | **Flow chain editor** — Activepieces-style vertical chain UI                                                                                                                                                                                                                                     | Build a flow entirely in the UI, no JSON editing                                                                              |
| 8   | **Computed columns** — reactive Rollup / Lookup / Template / If-Then-Else fields                                                                                                                                                                                                                 | Rollups update live across subscribed clients                                                                                 |
| 9   | **AI authoring loop** — chat agent generates/edits package artifacts; validate → ephemeral test site → generated tests → preview → approve                                                                                                                                                       | Prompt + CSV produces a tested, working app                                                                                   |

Deferred until the core earns them: typed AI steps in flows, human-in-the-loop resume links, "record enters view" triggers, per-step retries/error flows, marketplace/pieces framework for third-party steps.
