# Roadmap

Capabilities in build order. Each gets its own `docs/capabilities/N-name/` folder (research → spec → plan) written just-in-time — specs written far ahead of build are fiction. Later entries are provisional by design.

Resequenced 2026-07-09 after the first manual browser test of capability 3 (drift audit [#15](https://github.com/siraj-samsudeen/featherbase/issues/15)): sign-in and the UI shell pulled ahead of Excel import — a "user-facing" platform must be reachable by a user before it ingests their data. Authentication (4) split from authorization (7); pagination moved from Excel import into the UI capability (#16). Capabilities 1–3 folders keep their numbers; later entries renumbered.

| #   | Capability                                                                                                                                                                                                                                                                                                                 | The tracer-bullet test                                                                                                        |
| --- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| 1   | **Scaffold** — monorepo, React + Vite + TanStack Router, Convex, feather-testing-convex wired in, CI _that runs tests_                                                                                                                                                                                                     | A seeded integration test passes in CI                                                                                        |

| 3   | **Auto-generated UI** — create a DocType, get working grid, form, and detail views. Grid library needs re-research first — Glide Data Grid is unmaintained on npm since 2024-02 ([revalidation](research/revalidation-2026-07.md)); evaluate maintained forks, AG Grid, TanStack Table + Virtual                           | Build a working app with zero code                                                                                            |
| 4   | **Sign-in** — real authentication through the `requireUser` seam: Convex Auth, anonymous-first, sign-in/out in the shell; query-error states land here too ([#22](https://github.com/siraj-samsudeen/featherbase/issues/22), [#13](https://github.com/siraj-samsudeen/featherbase/issues/13))                              | Sign in from a real browser and run the zero-code loop — no identity injection, no hacks                                      |

Deferred until the core earns them: typed AI steps in flows, human-in-the-loop resume links, "record enters view" triggers, per-step retries/error flows, marketplace/pieces framework for third-party steps.
